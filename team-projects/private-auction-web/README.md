# 🔒 Secure Blind Auction System (보안 강화형 전자 봉인입찰 시스템)

본 시스템은 **비대칭 암호화(RSA-4096)**와 **봉투 암호화(Envelope Encryption)**를 결합하여,  
입찰 마감 전까지 **관리자·DBA·서버 운영자 누구도** 입찰가 평문을 알 수 없도록 설계된 고신뢰 경매 시스템입니다.

> **핵심 보장**
> - 마감 전: 입찰가는 암호화된 상태로만 저장됨
> - 마감 후: 관리자 승인이 있어야만 복호화 가능
> - 모든 입찰에 사용자 Ed25519 디지털 서명이 포함됨

## 2. 설치 및 환경 설정

### 2-1. 의존성 설치

```bash
go mod download
```

또는 Windows에서는 제공된 배치 파일 사용:

```bash
install_deps.bat
```

### 2-2. 환경 변수 설정 (`.env`)

프로젝트 루트의 `.env` 파일을 확인·수정하세요.

```env
# 데이터베이스 파일 경로
DB_PATH=db/blind_auction.db

# 루트 암호화 키 (AES-256용 — 반드시 정확히 32바이트(문자)여야 합니다!)
ROOT_KEK=your-32-byte-secret-key-here!!

# 실행 환경
ENV=development
```

> ⚠️ **`ROOT_KEK`는 반드시 정확히 32자여야 합니다.**  
> 짧거나 길면 경매 생성 시 `"invalid key size"` 오류가 발생합니다.

## 3. 첫 실행 — 초기 세팅 (반드시 먼저 하세요)

```bash

```

실행 결과 예시:

## 5. 역할별 화면 안내

### 🛡 ADMIN 로그인 시

```
관리자 대시보드
├── 화이트리스트 조회        ← 등록된 사용자 목록 확인
├── 화이트리스트 추가        ← 새 사용자에게 역할 부여
├── 화이트리스트 삭제        ← 사용자 권한 제거
├── 경매 관리 →             ← 경매진행자 화면으로 이동
└── 로그아웃
```

경매 관리 진입 시 AUCTIONEER 화면과 동일하게 경매 생성/마감/공개 가능.

---

### 🏷 AUCTIONEER 로그인 시

```
경매 관리 대시보드
├── 새 경매 생성             ← 제목, 시작/종료 시각 입력 → RSA 키쌍 자동 생성
├── 경매 목록 보기
│   └── 경매 선택 →
│       ├── [OPEN]    → 경매 마감하기 (OPEN → CLOSED)
│       ├── [CLOSED]  → 결과 공개하기 (CLOSED → REVEALED, 입찰가 복호화)
│       └── [REVEALED]→ 낙찰 결과 보기
└── 로그아웃
```

---

### 💰 BIDDER 로그인 시

```
입찰자 대시보드
├── (최초 로그인 시) Ed25519 서명 키 자동 발급 → keys/<username>_ed25519.pem 저장
├── 경매 목록 보기
│   └── 경매 선택 →
│       ├── [OPEN]    → 입찰가 입력 → 암호화 + 서명 후 제출
│       └── [REVEALED]→ 낙찰 결과 조회
└── 로그아웃
```

> ⚠️ `keys/<username>_ed25519.pem` 파일을 삭제하지 마세요.  
> 재발급 시 기존 입찰 서명 검증이 불가능해질 수 있습니다.

---

### 📝 처음 회원가입하는 경우

로그인 화면에서 `[ 회원가입 ]` 버튼을 선택하면 됩니다.  
화이트리스트에 등록된 username으로 가입하면 해당 역할이 자동 부여됩니다.  
화이트리스트에 없는 username은 가입 후 **GUEST** 권한으로 기능이 제한됩니다.

## 6. 전체 사용 흐름 예시

```
[1단계] 초기 세팅 (최초 1회)
  go run cmd/create_admin/main.go       ← 관리자 계정 생성

[2단계] 프로그램 실행
  go run cmd/blind-auction/main.go

[3단계] 경매 생성 (ADMIN 또는 AUCTIONEER 계정으로 로그인)
  → 새 경매 생성 → 제목 + 시작/종료 시각 입력
  → RSA-4096 키쌍 자동 발급 및 저장

[4단계] 입찰 (BIDDER 계정으로 로그인)
  → 경매 목록 → OPEN 경매 선택 → 입찰가 입력
  → AES-256-GCM 암호화 + Ed25519 서명 후 제출

[5단계] 경매 마감 및 결과 공개 (ADMIN 또는 AUCTIONEER)
  → 경매 목록 → 대상 경매 선택
  → [OPEN] "경매 마감하기" → CLOSED 상태로 전환
  → [CLOSED] "결과 공개하기" → 복호화 실행, REVEALED 상태로 전환
  → [REVEALED] "낙찰 결과 보기" → 낙찰자 및 전체 순위 확인

[6단계] 결과 확인 (BIDDER)
  → 경매 목록 → REVEALED 경매 선택 → 낙찰 결과 조회
```

## 7. 역할(Role) 설명

| 역할 | 설명 | 로그인 후 화면 |
|---|---|---|
| `ADMIN` | 시스템 전체 관리자. 화이트리스트 관리 + 경매 관리 가능 | 관리자 대시보드 |
| `AUCTIONEER` | 경매 진행자. 경매 생성·마감·공개 가능 | 경매 관리 대시보드 |
| `BIDDER` | 입찰자. 입찰 제출 및 결과 조회 가능 | 입찰자 대시보드 |
| `GUEST` | 화이트리스트 미등록 계정. 기능 제한 | 접근 불가 |

> 화이트리스트에 없는 사용자는 가입·로그인이 가능하지만 GUEST 처리됩니다.  
> 관리자가 화이트리스트에 추가해 주면 다음 로그인부터 역할이 적용됩니다.

## 8. 비밀번호 정책

회원가입 시 아래 규칙을 모두 만족해야 합니다:

```
✓ 8자 이상, 32자 이하
✓ 대문자 최소 1개  (A-Z)
✓ 소문자 최소 1개  (a-z)
✓ 숫자 최소 1개    (0-9)
✓ 특수문자 최소 1개 (!@#$%^&* 등)

예시: MyPass1!  ✅     mypassword  ❌
```

> 참고: 초기 관리자 비밀번호 `admin123!`은 위 규칙을 만족합니다.

## 9. 기술 스택 및 보안 설계

| 항목 | 내용 |
|---|---|
| **언어** | Go (Golang) |
| **데이터베이스** | SQLite (`db/blind_auction.db`) |
| **UI** | 터미널 TUI ([Bubble Tea](https://github.com/charmbracelet/bubbletea)) |
| **입찰 암호화** | AES-256-GCM (DEK) → RSA-4096 OAEP로 DEK 봉인 |
| **사용자 서명** | Ed25519 (입찰마다 사용자 개인키로 서명) |
| **키 관리** | RSA 개인키를 AES-256-GCM + ROOT_KEK로 래핑 후 DB 저장 |
| **인증** | 세션 토큰 (UUID, 24시간 유효) + 계정 잠금 (5회 실패 시 5분) |
| **감사 로그** | 모든 키 사용 및 데이터 접근 기록 |

### 보안 흐름 요약

```
[입찰 시]
입찰가
  → AES-256-GCM 암호화 (랜덤 DEK + 랜덤 Nonce)
  → DEK를 경매 RSA-4096 공개키로 암호화 (encrypted_DEK)
  → 전체 페이로드를 입찰자 Ed25519 개인키로 서명 후 제출

[결과 공개 시]
encrypted_DEK
  → ROOT_KEK로 보호된 RSA 개인키 복호화
  → DEK 복원 → AES-GCM 복호화 → 입찰가 평문 복원
```

## 🎬 사용 시나리오 (Usage Scenarios)

### 시나리오 A: 시스템 초기 설정 (관리자)
1. **관리자 생성**: `go run cmd/create_admin/main.go`를 실행하여 시스템의 첫 번째 관리자(`admin`)를 생성합니다.
2. **화이트리스트 등록**: `go run cmd/seed_whitelist/main.go`를 실행하여 `alice`, `bob`, `carol` 등 초기 멤버를 등록합니다.
3. **시스템 가동**: `go run cmd/blind-auction/main.go`를 실행하여 전체 시스템을 시작합니다.

### 시나리오 B: 경매 개최 및 관리 (경매인/관리자)
1. **로그인**: `carol` 또는 `admin` 계정으로 로그인합니다.
2. **경매 생성**: '새 경매 생성' 메뉴를 통해 제목(예: "빈티지 시계")과 마감 시각을 설정합니다.
3. **개인키 봉인**: 시스템은 경매용 RSA 키쌍을 생성하고, 개인키를 마스터 키(KEK)로 암호화하여 DB에 안전하게 보관합니다.
4. **마감 처리**: 경매 시각이 종료되면 경매 상세 화면에서 '경매 마감하기'를 선택합니다.
5. **결과 공개**: '결과 공개하기'를 선택하면 시스템은 보안 정책(KMSGuard)에 따라 개인키를 복호화하고, 모든 입찰 정보를 일괄 복호화하여 낙찰자를 결정합니다.

### 시나리오 C: 안전한 입찰 참여 (입찰자)
1. **로그인**: `alice` 또는 `bob` 계정으로 로그인합니다.
2. **서명키 발급**: 첫 로그인 시 시스템이 자동으로 Ed25519 서명 키를 생성하여 `keys/` 폴더에 저장합니다.
3. **경매 탐색**: '경매 목록 보기'에서 현재 진행 중인(OPEN) "빈티지 시계" 경매를 선택합니다.
4. **입찰 제출**: 입찰가(예: 1,500,000)를 입력합니다. 시스템은 이 값을 AES로 암호화하고, 경매의 RSA 공개키로 봉인한 뒤, 사용자의 서명과 함께 서버로 전송합니다.
5. **결과 확인**: 경매가 마감되고 상태가 REVEALED가 되면, 결과 화면에서 본인의 낙찰 여부와 최종 가격을 확인합니다.

### 시나리오 D: 보안 감사 및 로그 (보안 담당자)
1. **감사 로그 확인**: 관리자는 대시보드를 통해 모든 민감한 작업(키 생성, 복호화, 권한 변경)에 대한 감사 로그를 확인할 수 있습니다.
2. **로그 보호**: 모든 로그 메시지 내의 입찰가나 비밀번호 등은 `[MASKED]` 처리되어 내부 관리자에게도 노출되지 않습니다.
3. **무결성 검증**: 로그는 해시 연쇄(Hash Chaining)로 연결되어 있어, 누군가 로그를 변조하면 관리자 대시보드에서 즉시 탐지됩니다.
