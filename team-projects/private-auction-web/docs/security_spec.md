# Encryption Specification & Key Management

## 1. 암호화 알고리즘 및 표준
| 항목 | 표준 | 선택 이유 |
| :--- | :--- | :--- |
| **비대칭 암호화** | RSA-OAEP (4096-bit) | 입찰가의 기밀성 보장 및 선택 암호문 공격(CCA) 방어. |
| **디지털 서명** | Ed25519 (RFC 8032) | 사용자 및 서버의 서명 검증, 부인 방지. |
| **메시지 다이제스트** | SHA-256 | 데이터 커미트먼트, 감사 로그 무결성 및 공개키 지문 생성. |
| **대칭 암호화** | AES-256-GCM | KMS 봉투 암호화 및 입찰 데이터 암호화. |

## 2. 신뢰 체계 (Trust Mechanisms)

### 사용자 공개키 지문 (Fingerprint)
MITM 공격 방지를 위해 모든 사용자의 공개키는 SHA-256 해시값인 **지문(Fingerprint)**으로 관리됩니다.
- 사용자는 등록 시 지문을 확인하여 서버가 자신의 공개키를 변조하지 않았음을 증명할 수 있습니다.
- 모든 서명 검증은 등록된 지문과 매칭되는 공개키로만 수행됩니다.

### 입찰 영수증 (Digital Receipt)
서버는 입찰 수신 시 즉시 **디지털 영수증**을 발급합니다.
- **포맷**: `RECEIPT:BidID:AuctionID:CommitHash`
- **서명**: 서버의 전용 Ed25519 개인키로 서명됩니다.
- **용도**: 서버가 특정 입찰을 정상적으로 수신했음을 입증하는 법적/기술적 증거로 사용됩니다.

## 3. KMS 정책 및 다중 승인 (Quorum Policy)
경매용 RSA 개인키(Encrypted)는 KMS에 의해 엄격하게 보호됩니다.

### 해제 조건 (Unwrap Policy)
1.  **시간 정책 (Time-lock)**: `current_time > auction_end_time`. 마감 전 복호화는 KMS에 의해 물리적/논리적으로 거부됩니다.
2.  **다중 승인 (N-of-M Approvals)**: 사전에 지정된 다수의 관리자(Admin) 중 임계치 이상의 서명이 포함된 승인 토큰이 제출되어야 합니다.
    - 승인 토큰은 유효 기간(1시간)을 가집니다.
    - 동일 관리자의 중복 승인은 하나로 카운트됩니다.

## 4. 감사 로그 (Audit Log) 설계
모든 보안 이벤트는 SQLite `audit_logs` 테이블에 기록되며, 다음 정보를 포함합니다.
- **로그 연쇄 (Hash Chaining)**: 각 로그 엔트리는 이전 로그의 해시를 포함하여 로그의 부분 삭제 또는 변조를 방지함.
- **보안 이벤트**: `BID_SUBMITTED`, `KEY_UNWRAP_REJECTION`, `QUORUM_MET`, `RESULT_REVEALED`.

## 5. 데이터 폐기 정책
- **Zeroing Memory**: 메모리 내 평문 데이터(입찰가 등)는 사용 직후 0으로 채워져 소거됩니다.
- **KEK 회전**: 주기적으로 Root KEK를 갱신하며, 구버전 KEK로 암호화된 키는 폐기 정책에 따라 관리됩니다.
